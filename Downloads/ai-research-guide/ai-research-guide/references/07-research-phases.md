## 科研三个阶段概述

### 阶段划分原则
科研不是一蹴而就的，需要循序渐进：
- **第一阶段**：建立基础知识框架
- **第二阶段**：在实践中学习科研流程
- **第三阶段**：独立完成研究项目

### 阶段转换标准
每个阶段都有明确的完成标志：
- 第一阶段→第二阶段：完成基础课程，能够独立阅读论文
- 第二阶段→第三阶段：成功参与并完成一个完整的科研项目

### 时间规划参考
```
硕士/博士生典型时间线：
第1年：第一阶段（基础学习）
第2年：第二阶段（参与项目）
第3-4年：第三阶段（独立研究）
```

## 第一阶段：基础知识学习

### 阶段目标
**追求广度，不要求深度**：
- 了解3D Computer Vision的基础概念和算法
- 知道有哪些算法，知道大概原理
- 知道算法的应用场景
- 不要求掌握所有细节

### 学习资源

#### 核心课程
**《三维计算机视觉导论》**（Andreas Geiger教授）
- [课程链接](https://uni-tuebingen.de/fakultaeten/mathematisch-naturwissenschaftliche-fakultaet/fachbereiche/informatik/lehrstuehle/autonomous-vision/lectures/computer-vision/)
- **重点**：完成里面的作业
- **目标**：懂得做作业，也就懂课程内容了

#### 扩展课程
**CS231n（深度学习在计算机视觉的应用）**
- [课程链接](http://cs231n.stanford.edu/schedule.html)
- **注意**：此阶段可以不用做作业
- **策略**：只看课程视频和讲义
- **原因**：已经在具体实现某个论文，时间可能不够分配

#### 图形学课程（可选）
**GAMES101 - 现代计算机图形学入门**
- [课程链接](https://games-cn.org/intro-graphics/)
- 适合：对传统图形学知识感兴趣

**GAMES202 - 高质量实时渲染**
- 适合：学习渲染相关知识
- 讲师：闫令琪（讲得特别好）

**GAMES105 - 计算机角色动画基础**
- 适合：对motion synthesis/computer animation感兴趣
- 讲师：刘利斌（讲得特别好）

### 学习方法

#### 1. 建立GitHub仓库
```
目的：存放作业和学习记录

建议结构：
your-name-cv-course/
├── assignments/
│   ├── assignment1/
│   ├── assignment2/
│   └── assignment3/
└── notes/
    ├── camera-model.md
    ├── calibration.md
    └── ...
```

**好处**：
- 展示学习成果
- 方便后续回顾
- 建立代码库

#### 2. 理解而非死记
```
错误做法：
- 背诵公式
- 死记步骤
- 机械写代码

正确做法：
- 理解物理意义
- 掌握核心思想
- 知道什么时候用什么方法
```

#### 3. 问题求助策略
```
优先顺序：
1. Google搜索
2. 问New Bing
3. 问高年级学长/学姐
```

**注意事项**：
- 敢于提问，不要害羞
- 提问前先自己尝试解决
- 提供具体的问题背景和错误信息

### 阶段完成标志

**技术能力**：
- ✅ 能独立实现课程中的算法
- ✅ 能理解论文中的基本概念
- ✅ 能使用常见的深度学习框架

**知识体系**：
- ✅ 了解3D CV的主要分支
- ✅ 知道代表性工作和应用场景
- ✅ 建立初步的领域视野

## 第二阶段：参与科研项目

### 阶段目标
**追求深度，掌握某一篇论文的细节**：
- 构建某一个科研细分方向的算法基础
- 了解一篇论文是怎么做出来的
- 学习寻找科研问题、想Idea、做实验、写论文

### 三方面工作

#### 1. 选择研究方向
**选择原则**：
- 自己感兴趣
- 实验室有积累
- 有发展前景

**获取指导**：
- 请实验室学长/学姐帮自己定制具体学习计划

**学习计划示例**：
```
Week 1-2: 阅读5篇该方向的经典论文
Week 3-4: 复现其中1篇论文
Week 5-6: 分析该方向的Open Problems
Week 7-8: 尝试提出小改进并验证
```

#### 2. 系统学习深度学习
**推荐课程**：CS231n（深度学习在计算机视觉的应用）
- [课程链接](http://cs231n.stanford.edu/schedule.html)
- **策略**：不用做作业
- **方法**：只看课程视频和讲义
- **原因**：已经在具体实现某个论文，时间可能不够分配

**扩展学习**：
- 学习常用的工具、配置
- [工具配置文档](https://pengsida.notion.site/59569d7b66954578b21bf1dc6ea35776)（贺星毅整理）

#### 3. 旁听科研项目
**参与方式**：
- 旁听学长/学姐的科研项目
- 参加每周的讨论会
- 见识做科研具体是怎样的

**学习重点**：
- [Research Project的具体流程](https://pengsida.notion.site/research-project-b43507ef26d044bd888ac29f4736e116)
- 读论文、做实验的过程
- 团队协作方式

**逐步参与**：
1. 算法基础和实验能力足够时
2. 帮忙做Project的一些实验
3. 开始上手做科研项目

### 项目参与能力培养

根据[Research Project能力总结](https://pengsida.notion.site/research-project-b43507ef26d044bd888ac29f4736e116)，需要培养：

#### 1. 寻找重要科研问题
- 建立领域视野
- 识别未解决的问题
- 评估问题的重要性

#### 2. 想Idea
- 从已有工作中找灵感
- 跨领域迁移
- 分析实验中的意外发现

#### 3. 做实验
- 设计合理的实验
- 分析实验结果
- 调试和优化

#### 4. 写论文
- 清晰地表达思想
- 有逻辑地组织内容
- 准备图表和可视化

#### 5. 做Presentation
- 准备Slides
- 练习演讲
- 回答问题

### 阶段完成标志

**技术能力**：
- ✅ 能独立复现一篇论文
- ✅ 能做基本的实验和调试
- ✅ 能分析实验结果

**科研能力**：
- ✅ 了解科研项目的完整流程
- ✅ 能阅读和理解领域的最新论文
- ✅ 能参与讨论和提出建议

## 第三阶段：独立研究

### 阶段目标
**通过实践学习一篇论文是怎么做出来的**：
- 独立完成一个一作的Project
- 最好能完成一篇论文的投稿
- 在某个细分方向上有一定科研成果

### 前期准备：心理建设

**做科研和课程学习的区别**：
```
课程学习：
- 有明确的大纲和答案
- 按部就班就能得分
- 问题都有标准解法

科研探索：
- 没有标准答案
- 需要自己发现问题
- 经常遇到失败和挫折
```

**需要具备的心态**：
- 接受失败是常态
- 保持耐心和毅力
- 不断学习和调整
- 主动沟通和求助

### 必需的科研能力

根据[如何培养自己的科研能力](https://pengsida.notion.site/idea-da6ce171c13846b7a7ffaa7473ffa6ea)：

#### 1. 想Idea的能力
**培养方法**：
- 大量阅读文献，建立领域视野
- 关注实验中的意外发现
- 跨领域学习和迁移
- 批判性思考

**实战练习**：
- 每读一篇论文，问三个问题：
  - 核心假设是什么？
  - 哪些场景下会失效？
  - 有没有更简洁的方式？

#### 2. 写作能力
**培养方法**：
- 读优秀论文，学习写作风格
- [练习写论文](https://pengsida.notion.site/c13c7e52aab64c1a8e3576b97fcb9851)
- 使用[论文写作模板](https://pengsida.notion.site/c1a22465a0fa4b15a12985223916048e)
- 学习[高水平科研工作者的写作经验](https://pengsida.notion.site/74aef88b9187439fa4e301704f6eb49a)

**实践途径**：
- 写论文初稿
- 写Rebuttal
- 做学术报告Slides
- 写Demo和说明

#### 3. 实验能力
**培养方法**：
- 写代码做实验
- [分析实验不work的原因](https://pengsida.notion.site/1aee6e718de6472f834d13da8f4ff097)
- [写实验记录](https://pengsida.notion.site/caf34717f4c046c69ee7e14ea953c46f)
- 学习可视化工具

**关键技能**：
- 调试和Debug
- 实验设计和优化
- 结果分析和可视化
- 可复现性保证

### Top PhD的能力模型

#### 核心能力：设定长远目标

**什么是长远目标**：
- 具有重要的科学价值
- 具有实际应用价值
- 在实际应用中寻找真正有价值的科学问题

**如何设定长远目标**：
```
Step 1: 建立领域视野
- 了解领域的发展趋势
- 识别未解决的挑战

Step 2: 发现有价值的科学问题
- 什么问题真正重要？
- 什么问题有应用价值？
- 什么问题有理论价值？

Step 3: 细化Roadmap
- 长期目标分解为几个子问题
- 每个子问题对应一篇论文
- 论文之间有逻辑关系
```

#### 五大能力详解

**1. 寻找重要的科研问题**
- **特征**：能识别真正有价值的问题
- **培养**：
  - 大量阅读，建立领域视野
  - 关注实际应用需求
  - 思考理论意义
- **体现**：做的论文围绕解决一个科研目标

**2. 提出解决方案**
- **特征**：能针对问题提出有效的解决方案
- **培养**：
  - 掌握多种技术方法
  - 理解方法适用的场景
  - 能够组合和创新
- **体现**：方法有创新性且有效

**3. 做实验**
- **特征**：能设计合理的实验并验证想法
- **培养**：
  - 掌握实验工具
  - 理解统计方法
  - 具备分析能力
- **体现**：实验充分，结果可信

**4. 写论文**
- **特征**：能清晰、有逻辑地表达研究成果
- **培养**：
  - 多读多写
  - 学习优秀论文
  - 不断修改和润色
- **体现**：论文被顶会接收

**5. 做Presentation**
- **特征**：能有效地口头传达研究成果
- **培养**：
  - 练习演讲
  - 准备高质量Slides
  - 回答问题
- **体现**：报告获得好评

#### 博士论文的连贯性

**理想的博士研究**：
```
长期目标：解决X领域中的Y问题

Roadmap:
论文1: 理解问题，提出初步方法
论文2: 改进方法，提升性能
论文3: 扩展应用，展示泛化性
论文4: 理论分析，深入理解

最终成果：在Y问题上取得系统性突破
```

**体现方式**：
- Demo要很酷（展示效果）
- 故事要完整（逻辑清晰）
- 影响要持久（有长远价值）

## 心态调整

### 接受现实

**科研不是直线**：
```
理想路径：
Idea → 实验 → 结果 → 论文 → 接收

实际路径：
Idea → 实验1失败 → 实验2部分成功 → 分析
→ 改进Idea → 实验3成功 → 论文
→ Rebuttal → 修改 → 接收
```

**螺旋上升**：
- 不是每次都在进步
- 有时候需要回头
- 每次循环都在提升

### 应对挫折

**常见的挫折**：
- 实验多次失败
- 论文被拒
- 撞车（别人先发表了类似工作）
- 看不到进展

**应对策略**：
1. **接受失败**：
   - 失败排除错误答案，也有价值
   - 分析失败的过程是学习过程

2. **保持记录**：
   - 记录所有实验（成功的和失败的）
   - 记录思考和假设
   - 定期回顾

3. **寻求帮助**：
   - 与导师讨论
   - 与同学交流
   - 不要孤军奋战

4. **调整预期**：
   - 不要期望立即成功
   - 给自己时间成长
   - 享受探索过程

### 保持动力

**设定小目标**：
```
Week目标：
- 阅读3篇论文
- 复现1个方法
- 提出Idea
```

**庆祝小胜利**：
- 成功复现一篇论文
- 实验有改进（即使很小）
- 论文被接收（即使是小会议）

**看到进步**：
- 定期回顾自己的成长
- 对比几个月前的自己
- 认识到积累的重要性

## 常见问题与解答

### Q1: 第一阶段需要多久？
**A**: 通常6-12个月，取决于基础。目标不是"学完所有知识"，而是"建立框架，能够继续学习"。

### Q2: 可以跳过第一阶段直接做项目吗？
**A**: 不推荐。基础不扎实，做项目会很吃力，而且容易走弯路。

### Q3: 第二阶段需要参与几个项目？
**A**: 至少1个完整的。重点是学习流程，不是数量。

### Q4: 如果第三阶段没有发论文怎么办？
**A**: 正常。科研需要积累。重要的是学习过程，不是结果。

### Q5: 如何知道自己准备好了进入下一阶段？
**A**: 参考每个阶段的"完成标志"。不确定时，可以问导师或学长。

### Q6: 如果对选定的方向不感兴趣怎么办？
**A**: 尽早与导师沟通。可以调整方向，但要有充分理由。

### Q7: 如何平衡课程学习和科研？
**A**: 第一阶段以课程为主，第二、三阶段以科研为主。合理安排时间。

### Q8: 需要多久才能独立发顶会？
**A**: 因人而异。通常需要2-3年的积累。不要急于求成。

## 阶段转换检查清单

### 第一阶段→第二阶段
在进入第二阶段之前，检查：
- [ ] 完成了核心课程的学习？
- [ ] 能独立实现课程中的算法？
- [ ] 建立了基础的领域知识框架？
- [ ] 能理解论文中的基本概念？
- [ ] 有GitHub仓库存放学习记录？

### 第二阶段→第三阶段
在进入第三阶段之前，检查：
- [ ] 参与过至少1个完整的科研项目？
- [ ] 能独立复现一篇论文？
- [ ] 理解科研项目的完整流程？
- [ ] 能做基本的实验和分析？
- [ ] 与导师讨论过独立研究计划？

### 第三阶段：独立研究
在开始独立研究之前，检查：
- [ ] 有明确的研究方向和目标？
- [ ] 制定了详细的Roadmap？
- [ ] 有了初步的Idea？
- [ ] 准备好面对失败和挫折？
- [ ] 与导师确认了研究计划？

## 总结

### 核心原则
1. **循序渐进**：按阶段推进，不要跳步
2. **实践为主**：在实践中学习，不只是读书
3. **主动沟通**：寻求导师和同学的帮助
4. **保持记录**：记录所有学习过程
5. **享受探索**：科研是探索未知的过程

### 关键成功因素
- **建立领域视野**：多读论文，了解全局
- **培养批判思维**：不盲从，多思考
- **动手实践**：不只是理论，要动手
- **接受失败**：失败是学习过程
